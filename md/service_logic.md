# 서비스 기획 및 분석 로직 (Eye-Brow Architect)

이 문서는 **'Eye-Brow Architect'**의 핵심 비즈니스 로직인 AI 분석 프로세스, 시각화 기술, 그리고 매칭 알고리즘을 상세히 다룹니다.

---

## 1. 지능형 상담 및 분석 파이프라인
사용자의 니즈를 파악하고 가이드를 생성하는 전체 프로세스입니다.

1. **상담 및 정보 수집 (Lex & Transcribe):**
   - 사용자는 텍스트 또는 **음성(Transcribe)**을 통해 자신의 TPO(결혼식, 면접 등)를 설명합니다.
   - 렉스(Lex)는 사용자의 의도를 파악하고 정면 얼굴 사진 업로드를 요청합니다.
2. **특징점 추출 (Rekognition):**
   - 업로드된 사진에서 눈매, 눈썹 위치, 얼굴 윤곽 등 30개 이상의 주요 좌표를 추출합니다.
3. **전문가 분석 로직 (Python/EKS):**
   - 추출된 좌표를 기반으로 $1:1.5$ 황금비율 계산 및 5대 얼굴형을 판별합니다.
4. **개인화 매칭 (Matching Logic):**
   - 판별된 얼굴형과 DB에 등록된 메이크업 스타일 간의 **'상관계수'**를 계산하여 최적의 스타일을 추천합니다.
5. **결과 저장 및 보고:**
   - 모든 분석 데이터는 RDS에 저장되며, 사용자에게는 맞춤형 가이드 텍스트와 추천 이미지가 제공됩니다.

---

## 2. 가상 시뮬레이션 (Virtual Overlay) ⭐
Canvas API를 활용하여 정교한 메이크업 가이드를 시각적으로 제공합니다.

- **실시간 드로잉:** 사용자의 원본 사진 위에 분석된 좌표를 기반으로 눈썹 형태(아치형, 일자형 등)와 아이라인 가이드를 실시간으로 그립니다.
- **투명도 및 레이어링:** 가이드 라인의 투명도를 조절하여 실제 메이크업 시 참고하기 쉬운 자연스러운 오버레이를 구현합니다.
- **비포/애프터 비교:** 원본 사진과 가이드가 적용된 사진을 동시에 노출하여 시각적 변화를 직관적으로 확인하게 합니다.

## 3. 나혜님의 핵심 분석 및 매칭 로직
전공 지식과 수학적 계산을 결합한 고도화된 추천 시스템입니다.

- **얼굴형-스타일 상관계수:**
   - 각 얼굴형(달걀, 둥근, 긴, 각진, 역삼각형)마다 어울리는 메이크업 요소들의 가중치를 설정합니다.
   - 사용자의 진단 데이터와 메이크업 태그 간의 매칭 점수를 산출하여 '궁합 점수'로 표시합니다.
- **복합 처방 예시:** 
   - **Case:** 역삼각형 얼굴 + 뾰족한 턱선 ➡️ 시선을 위로 분산시키기 위해 눈썹 산을 부드럽게 강조하고, 눈썹 꼬리를 약간 길게 설계하여 하관의 날카로움을 보완.
- **알고리즘 유연성:** 관리자 대시보드(Admin Panel)를 통해 트렌드에 맞춰 매칭 가중치와 로직을 실시간으로 업데이트할 수 있습니다.

## 5. 사용자 참여형 커뮤니티 및 추천 시스템 (New) ⭐ [cite: 2026-03-04]
기존의 권한 분리 구조를 탈피하여, 모든 사용자가 메이크업 데이터를 생성하고 공유할 수 있는 커뮤니티 기반 구조로 변경되었습니다.

1. **커뮤니티 기반 CRUD:**
   - 모든 사용자는 자신만의 메이크업 스타일을 등록, 수정, 삭제할 수 있습니다.
   - 공유된 데이터는 전체 사용자의 '추천 풀(Pool)'로 활용됩니다.
2. **AI 분석 기반 맞춤형 추천:**
   - 상담 페이지의 AI는 커뮤니티에 축적된 대량의 데이터 중, 사용자의 얼굴 분석 결과(얼굴형, 특징점)와 가장 유사한 항목을 필터링합니다.
   - **매칭 알고리즘:** 사용자의 특징 벡터와 스타일 태그 간의 거리 점수를 계산하여 순차적으로 나열합니다.
3. **Paging & Next Recommendation:**
   - 한 번에 하나의 스타일만 추천하는 것이 아니라, 사용자가 만족할 때까지 '다음 추천(Next)' 페이지를 넘기며 탐색할 수 있도록 설계합니다.
   - 백엔드는 추천 리스트를 넘겨줄 때 효율적인 응답을 위해 **페이징 처리(Paging)**를 수행합니다.

## 6. 데이터 지표 및 개인화 최적화 (Advanced) ⭐ [cite: 2026-03-04]
사용자 경험 향상과 데이터 기반 추천을 위해 새로운 메트릭과 플래그를 도입합니다.

1. **인기 지표 (Engagement Metrics):**
   - **`like_count` (좋아요):** 사용자가 특정 스타일을 선호하는 지표.
   - **`view_count` (조회수):** 스타일이 노출되거나 클릭된 횟수.
   - **추천 로직 반영:** 추천 리스트 생성 시, 얼굴 분석 매칭 점수가 동일하다면 `like_count`와 `view_count`가 높은 '인기 스타일'을 우선적으로 배치합니다.
2. **스크랩 기능 (Scrap):**
   - 사용자가 마음에 드는 스타일을 자신의 보관함에 저장할 수 있으며, 이는 `like_count`에 가중치를 부여하는 요소가 됩니다.
3. **최신 분석 식별 (`is_latest`):**
   - 한 사용자가 여러 번 사진을 등록하여 분석할 수 있으므로, DB의 여러 분석 레코드 중 가장 최근의 유효한 데이터를 구분하기 위해 `is_latest` 플래그(Boolean)를 사용합니다.
   - AI 상담 시 항상 `is_latest=true`인 분석 결과를 기준으로 추천을 시작합니다.

## 7. 로컬 API 설계 가이드 (Backend API Design) [cite: 2026-03-04]
인기 스타일 추천 및 스크랩 기능을 위한 핵심 API 구조입니다.

- **`GET /api/styles/recommend?page=0&size=5&sort=popular`**
  - **설명:** 현재 사용자의 최신 분석(`is_latest=true`) 결과와 매칭되는 스타일을 인기순(`like_count` DESC)으로 페이징하여 반환합니다.
- **`POST /api/styles/{styleId}/scrap`**
  - **설명:** 특정 스타일을 사용자의 스크랩 리스트에 추가하고 해당 스타일의 `like_count`를 1 증가시킵니다.
